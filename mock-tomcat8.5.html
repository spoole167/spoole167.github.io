Critical Configuration Requirement

This vulnerability only affects systems where readonly="false" is set on the Default Servlet. The default Tomcat configuration has writes disabled, meaning most default installations are not vulnerable.

Vulnerability Privately Reported

Apache Tomcat security team receives private report of a vulnerability involving partial HTTP PUT requests and the default file servlet.

Technical Details

The flaw was identified as affecting Tomcat's file upload handling when writes are enabled (non-default configuration). The vulnerability allows path equivalence exploitation where filenames containing internal dots can bypass security checks.
